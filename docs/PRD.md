# Daily Fortune 产品需求文档

## 1. 产品概述
- **产品名称**：Daily Fortune
- **产品定位**：一个轻量、沉浸式的 H5 刮刮卡网页，用户通过刮开金色遮罩查看当日运势。
- **目标**：用低门槛交互提升停留时长与分享意愿，适合作为活动页、引流页或轻娱乐页面。

## 2. 目标用户
- 喜欢轻量互动体验的移动端用户
- 需要节日营销、品牌互动页的运营团队
- 希望快速上线静态活动页的前端团队

## 3. 用户场景
- 用户打开页面后看到“今日运势”主题页
- 用户通过鼠标或手指刮开奖励层
- 刮开后直接呈现下方内容，无须进度校验
- 用户点击“Share My Fortune”分享结果
- 用户点击“Try Again Tomorrow”重新开始体验

## 4. 核心功能需求

### 4.1 页面展示
- 页面需展示标题、说明文案、运势内容区和两个操作按钮
- 页面风格需保持神秘、占卜感与移动端友好布局

### 4.2 刮刮卡交互
- 使用 `canvas` 渲染遮罩层
- 支持鼠标和触摸事件
- 支持平滑且连续的刮开交互（使用线段绘制而非独立散点）

### 4.3 运势内容
- 展示今日幸运星级
- 展示提示语、每日格言等固定内容
- 运势内容应在未刮开时被遮罩覆盖

### 4.4 分享能力
- 点击分享按钮时优先调用浏览器原生分享能力
- 如果设备不支持原生分享，则退化为复制文案到剪贴板
- 分享结果需要有用户可见反馈

### 4.5 重新开始
- 点击“Try Again Tomorrow”后刷新页面，恢复初始刮刮卡状态

## 5. 非功能需求
- **性能**：首屏资源轻量，保证静态站点可快速加载
- **兼容性**：兼容桌面端与主流移动浏览器
- **可访问性**：交互按钮需要保留明确文案，分享反馈应支持状态播报
- **可测试性**：关键逻辑需可单元测试，构建与测试需可在 CI 自动执行

## 6. 验收标准
- 页面可正常构建并部署为静态资源
- 用户可在鼠标与触摸环境中完成刮开操作
- 刮开过程无遮罩碎片和缝隙
- 分享按钮在支持场景下触发系统分享，不支持时复制文案
- 自动化测试与构建可在 GitHub Actions 中通过
- 测试报告可在仓库中查看并作为 CI 产物归档

## 7. 风险与后续建议
- 当前运势内容为静态文案，后续可接入动态内容源
- 当前无埋点统计，后续可增加分享点击与完成率监控
- 后续可增加多语言、运势模板切换和日期驱动逻辑
